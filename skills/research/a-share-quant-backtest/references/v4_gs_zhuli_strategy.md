# v4 多票组合策略 — GS+主力雷达（迭代历史完整记录）

## 策略迭代历程

### v1 — 初始版本
- **买入**：GS在G区间 + 主力线上穿零轴 + 散户线下降
- **卖出**：主力线连续2日下降
- **结果**(5只)：总收益 +9.3%，交易17次
- **问题**：卖出信号过密（~30%交易日触发），买入信号稀缺

### v2 — 放宽买入+收紧卖出
- **买入**：去掉散户线下降条件
- **卖出**：2日→4日连降
- **结果**(5只)：总收益 +230%，Sharpe 1.79，交易62次
- **核心发现**：连降天数是最敏感的调参轴心

### v3 — 扩展到99只
- 同v2参数，扩展到99只自选股
- **结果**：总收益 +117%，最大回撤 -35%，Sharpe 1.02，交易117次
- **暴露问题**：大样本下五种亏损模式浮现

### v4 — 诊断亏损并修复
- **三大根因**：
  1. 信号簇聚：卖出后立刻重新买入同一只，反复割肉
  2. 高位接盘：主力线上穿时已涨很多，zhuli动辄+100~200
  3. 卖出太晚：4连降触发时已从峰值回撤15-30%
- **修复方案**：
  1. 个股卖出后10天冷静期（不买回）
  2. zhuli上穿时 < 60 过滤 + 恢复散户线下降条件
  3. 4连降→3连降 + 新增-8%硬止损
- **结果**(99只)：总收益 +62.5%，最大回撤 -25%，Sharpe 0.89，交易153次

## 当前策略（v4最终参数）

### 买入信号 (AND)
1. GS 处于 G 信号区间 (`gs_bull_market=True` 或 `gs_g_point=True`)
2. 主力线 上穿零轴（昨日≤0 且 今日>0）
3. 主力线上穿时幅度 **< 60**（防止高位接盘）
4. 散户线下降（`sanhu[today] < sanhu[yesterday]`，过滤假突破）

### 卖出信号（任一触发）
1. **-8% 硬止损**（优先级最高，无条件执行）
2. 主力线连续 **3日** 下降
3. **利润回调**：盈利>10%时不卖，从峰值回调5%时卖

### 组合管理
- 每只 1/3 资金，最多同时持有 3 只
- 买入：信号日次日**开盘**市价
- 卖出：信号当日**收盘**市价（含止损）
- 排序：按主力线上涨幅度（今日-昨日，绝对值）降序
- 冷静期：卖出某股后 **10天** 内不买回该股
- 可空仓、可不满仓

## 参数敏感性分级

| 参数 | 敏感度 | 说明 |
|------|--------|------|
| 连降天数 | ⭐⭐⭐⭐⭐ | 2→3→4，收益率变化巨大。核心参数 |
| zhuli阈值 | ⭐⭐⭐⭐ | 60→80 可能找回被误杀的好信号 |
| 散户线条件 | ⭐⭐⭐ | 加上后买入信号锐减，过滤效果强 |
| 冷静期天数 | ⭐⭐ | 10天适中，太短无效、太长浪费机会 |
| 止损比例 | ⭐⭐ | -8% 标准值 |

## 亏损诊断方法论

当某只股票策略亏损时，按以下步骤诊断：

1. **检查买入持有收益**：股票本身是牛是熊？
   - 熊股（买入持有为负）→ 策略亏损可以接受（保护本金）
   - 牛股（买入持有为正但策略亏）→ 策略有问题

2. **逐笔交易分析**：对每笔买卖检查
   - 买入时 zhuli 值（是否高位接盘）
   - 持仓期间峰值 PnL（有没有赚钱机会）
   - 卖出时 zhuli 值（是否深度负值）
   - 持有天数（是否短期反复割肉）

3. **模式识别**：
   - 同一日期多笔卖出 → 信号簇聚
   - zhuli 买入时 >100 → 高位接盘
   - 峰值PnL很高但最终亏损 → 卖出太晚

诊断脚本参考：`analyze_losers.py`（项目根目录）
