## 已知数据格式

### 1. 同花顺 hd1.0 格式 (本技能参考文件详细描述)

**文件标识：** 前6字节为 `hd1.0\0` (0x68 0x64 0x31 0x2e 0x30 0x00)

**结构：**
- 6B: 魔数 `hd1.0\0`
- 4B: 记录数 (int32 LE)
- ~170B: 索引区 (元数据)
- 数据区: 每条记录固定大小

**记录大小：**
| 类型 | 记录大小 | 起始偏移 |
|------|---------|---------|
| 指数 | 164 字节 | 0xB4 (180) |
| 个股 | 176 字节 | 0xC0 (192) |

**单条记录布局 (前28B)：**
| 偏移 | 字段 | 格式 | 说明 |
|------|------|------|------|
| 0-3 | 日期 | int32 LE | YYYYMMDD |
| 4-6 | 开盘价 | int24 LE / 10000 | 分隔符0xC0在字节7 |
| 8-10 | 最高价 | int24 LE / 10000 | 分隔符0xC0在字节11 |
| 12-14 | 最低价 | int24 LE / 10000 | 分隔符0xC0在字节15 |
| 16-18 | 收盘价 | int24 LE / 10000 | 分隔符0xC0在字节19 |
| 20-23 | 成交额 | int32 LE | 单位：元 |
| 24-27 | 成交量 | int32 LE | 单位：股(个股) 或手(指数) |

**关键发现：** 价格存储在3字节LE整数中，除以10000得到实际价格。

### 2. 通达信标准 .day 格式

**结构：** 每条记录固定32字节，无文件头。

| 偏移 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 0-3 | 日期 | int32 | YYYYMMDD |
| 4-7 | 开盘价 | float32 / 100 | |
| 8-11 | 最高价 | float32 / 100 | |
| 12-15 | 最低价 | float32 / 100 | |
| 16-19 | 收盘价 | float32 / 100 | |
| 20-23 | 成交额 | int32 | 元 |
| 24-27 | 成交量 | float32 / 100 | 手 |
| 28-31 | 保留 | int32 | |

Python解析：
```python
import struct
with open(file, 'rb') as f:
    buf = f.read()
for i in range(len(buf) // 32):
    a = struct.unpack('IIIIIfII', buf[i*32:(i+1)*32])
    date = str(a[0])  # YYYYMMDD
    open_ = a[1] / 100.0
    high = a[2] / 100.0
    low  = a[3] / 100.0
    close = a[4] / 100.0
    amount = a[5]
    volume = a[6] / 100.0
```

**注意：** 通达信 .day 文件是**裸数据**，没有文件头。而同花顺 hd1.0 格式的文件有6字节头部 `hd1.0\0`。可通过检查前6字节区分两种格式。

## 已知陷坑

### 1. 不要用通达信的32字节解析同花顺文件
同花顺 hd1.0 格式有6字节头部和索引区，记录大小为164/176字节而非32字节。

### 2. 价格是除以10000不是100
同花顺格式使用 int24 ÷ 10000，通达信格式使用 float32/÷100。

### 3. 每条价格后面有0xC0分隔符
这是同花顺格式的特点，在解析时需要跳过，不要把它们当成价格数据。

### 4. 索引区不是数据区
hd1.0 格式在头部后有一个索引区（约170字节），需要跳过才能找到实际数据。

### 5. 文件是倒序存储
.day 文件中的记录通常是按时间倒序排列的（最近的在前面），解析后需要按日期排序。
