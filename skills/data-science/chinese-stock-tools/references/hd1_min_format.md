# 同花顺 hd1.0 分钟数据(.min) 格式规范

## 文件头

与 `.day` 文件相同：`hd1.0\0` (6字节) + count (4字节 LE int32)

## 索引区

- 范围：字节 10 ~ 215（约206字节）
- 结构与 `.day` 文件类似，包含 2 字节/4 字节交错编码的元数据

## 数据记录

每条记录占据 **152 字节** 的槽位，其中有效数据为 **48 字节**，剩余为 0xFF 填充。

### 48 字节记录结构

| 偏移 | 字段 | 说明 |
|------|------|------|
| 0-3 | seq | 顺序号 (int32 LE) |
| 4-7 | open | 价格字段，见下方格式说明 |
| 8-11 | high | 同上 |
| 12-15 | low | 同上 |
| 16-19 | close | 同上 |
| 20-23 | 字段5 | 用途不明（可能是累计成交相关） |
| 24-27 | volume | 成交量 (int32 LE) |
| 28-31 | amount | 成交额 (int32 LE) |
| 32-35 | 字段8 | 用途不明 |
| 36-39 | trades | 成交笔数 (int32 LE) |
| 40-43 | 字段10 | 用途不明 |
| 44-47 | 字段11 | 用途不明 |

### 两种价格编码格式

自动检测，无需手动指定：

#### 格式 A：int24 (3字节 + 0xc0 分隔符)

- **价格**: 3字节 LE 无符号整数 ÷ **10000**
- **字节 7, 11, 15, 19**: 固定为 `0xc0`（分隔符）
- 适用于：B股指数等小数值（如 上证B指 ~260）
- 举例：`48 d3 2a c0` → 0x2ad348 = 2806600 → ÷10000 = **280.66**

#### 格式 B：int32 (4字节)

- **价格**: 4字节 LE 无符号整数 ÷ **1000000**
- 字节 7, 11, 15, 19 **不能**为 0xc0
- 适用于：上证指数等大数值（如 上证指数 ~3262）
- 举例：`18 d6 6e c2` → 0xc26ed618 = 3262054936 → ÷1000000 = **3262.05**

### 格式检测算法

```
1. 检查字节 7, 11, 15, 19 是否全是 0xc0 → int24
2. 检查以上位置全不是 0xc0，且 4个价格值接近(最大差 < 50%)
   且均落在 [1e6, 1e10] 范围内 → int32
3. 否则 → unknown
```

### 记录的连续性

记录槽位间距固定为 **152 字节**。有效记录占前 48 字节，其后 104 字节为 0xFF 填充。
文件末尾的无效槽位（数据结束后）全部为 0xFF。

## 数据起始偏移

数据区通常在偏移 **320 (0x140)** 开始。自动探测逻辑：
1. 遍历偏移 160-800（步长 4）
2. 对每个候选偏移，检查 48 字节块的格式（int24/int32）
3. 验证下一个 152 字节后的槽位格式一致
4. 匹配则返回该偏移

## 与日线数据的区别

| 特性 | `.day` 日线 | `.min` 分钟 |
|------|------------|------------|
| 有效记录大小 | 164B(指数) / 176B(个股) | 48B |
| 记录间距 | 连续 | 152B (+104B填充) |
| 价格格式 | 仅 int24 | int24 或 int32 |
| 时间标识 | YYYYMMDD (int32) | seq 顺序号 |
| 数据起始 | 180(指数) / 192(个股) | 320 |
| 数据库表 | daily_kline | minute_kline |
