# 同花顺 hd1.0 日线数据格式详细规范

> 逆向工程结果，基于 `/Users/yellow/Documents/同花顺指标/同花顺原数据/shase/day/` 目录下多个 .day 文件验证。
> 验证文件：1A0003 (指数), 600350, 688519, 603459 (个股)

## 1. 文件结构总览

```
+------------------+-----------+
| 魔数 "hd1.0\0"    | 6 字节    |
+------------------+-----------+
| 记录数 (int32 LE) | 4 字节    |
+------------------+-----------+
| 索引区           | ~170 字节 |
+------------------+-----------+
| 数据区           | N * RecSize |
+------------------+-----------+
```

### 1.1 魔数
前6字节固定为 `68 64 31 2e 30 00`，即 ASCII `hd1.0\0`。

### 1.2 记录数
紧接魔数后的4字节 int32 LE。例如：
- 1A0003.day: `94 00 00 00` = 148
- 600350.day: `8b 02 00 00` = 651

### 1.3 索引区
从偏移10开始到数据区起始。内容为元数据，包含`0x00 0x04 0x?? 0x70`模式的条目序列，以及：
- 指数文件(1A0003)：索引区 170 字节，数据从偏移 180 (0xB4) 开始
- 个股文件(600350)：索引区 182 字节，数据从偏移 192 (0xC0) 开始

## 2. 数据记录格式

### 2.1 核心记录 (前28字节, 指数和个股通用)

| 偏移 | 长度 | 字段 | 类型 | 取值示例 | 说明 |
|------|------|------|------|---------|------|
| 0 | 4 | 日期 | int32 LE | `0x013501df` = 20251103 | YYYYMMDD |
| 4 | 3 | 开盘价 | int24 LE | `0x010f7c` = 69500 ÷10000 = 6.95 | 3字节小端整数 |
| 7 | 1 | 分隔符 | uint8 | `0xc0` | 固定值，跳过 |
| 8 | 3 | 最高价 | int24 LE | `0x011814` = 71700 ÷10000 = 7.17 | 同上 |
| 11 | 1 | 分隔符 | uint8 | `0xc0` | 固定值 |
| 12 | 3 | 最低价 | int24 LE | `0x010e50` = 69200 ÷10000 = 6.92 | 同上 |
| 15 | 1 | 分隔符 | uint8 | `0xc0` | 固定值 |
| 16 | 3 | 收盘价 | int24 LE | `0x0117b0` = 71600 ÷10000 = 7.16 | 同上 |
| 19 | 1 | 分隔符 | uint8 | `0xc0` | 固定值 |
| 20 | 4 | 成交额 | int32 LE | `0x10d17e69` = 282164841 | 单位：元 |
| 24 | 4 | 成交量 | int32 LE | `0x0126c7b5` = 19318709 | 单位：指数字/股 |

### 2.2 价格读取公式 (Python)

```python
def read_int24_le(data, offset):
    """从3字节小端数据读取整数"""
    return data[offset] + (data[offset+1] << 8) + (data[offset+2] << 16)

# 示例：读取开盘价
open_raw = read_int24_le(buf, offset + 4)   # 如 69500
open_price = open_raw / 10000.0              # 如 6.95
```

### 2.3 记录大小

| 文件类型 | 记录大小 | 数据起始偏移 | 说明 |
|---------|---------|-------------|------|
| 指数文件 (1Axxxx) | 164 字节 | 0xB4 (180) | 上证系列指数 |
| 个股文件 (6xxxxx) | 176 字节 | 0xC0 (192) | 沪市/深市个股 |

### 2.4 排序方式
原始.day文件按时间**倒序**存储（最新的在前面），解析后应正序排序。

## 3. 自动探测算法

由于指数和个股的记录大小不同，需要自动探测：

```python
def find_data_area(buf):
    """遍历可能的记录大小(164/176)，找连续5条有效日期"""
    for rec_size in [164, 176]:
        for off in range(10, min(500, len(buf)-4), 2):
            val = int.from_bytes(buf[off:off+4], 'little')
            if 19900101 <= val <= 20270630:
                if off + rec_size + 4 <= len(buf):
                    val2 = int.from_bytes(buf[off+rec_size:off+rec_size+4], 'little')
                    if 19900101 <= val2 <= 20270630 and val2 != val:
                        # 再验证3条
                        ok = True
                        for k in range(2, 5):
                            off3 = off + rec_size * k
                            if off3 + 4 <= len(buf):
                                v3 = int.from_bytes(buf[off3:off3+4], 'little')
                                if v3 < 19900101 or v3 > 20270630:
                                    ok = False
                                    break
                        if ok:
                            return off, rec_size
    return None, None
```

## 4. 与通达信标准 .day 格式对比

| 特性 | 同花顺 hd1.0 | 通达信标准 .day |
|------|-------------|----------------|
| 文件头 | `hd1.0\0` (6B) | 无文件头 |
| 记录大小 | 164(指数)/176(个股) | 32 (通用) |
| 价格类型 | int24 LE | float32 |
| 价格除数 | 10000 | 100 |
| 数据起始 | 偏移 180/192 | 偏移 0 |
| 排序 | 倒序（新→旧） | 未知（可能倒序） |
